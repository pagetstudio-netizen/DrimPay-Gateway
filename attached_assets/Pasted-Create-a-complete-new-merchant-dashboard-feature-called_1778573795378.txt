Create a complete new merchant dashboard feature called “Pay with QR” inside DrimPay.

IMPORTANT:
- Do NOT modify the existing payment engine.
- Do NOT modify the automatic credit system.
- Do NOT modify the current payin logic, wallets, webhook flow, transaction system, or payment links system.
- This QR system must reuse the existing payin infrastructure and existing checkout page.

════════════════════════════
FEATURE OVERVIEW
════════════════════════════

Add a new sidebar menu item:

→ Pay with QR

The purpose of this feature is to allow merchants to generate static QR payment codes linked to their DrimPay account.

When customers scan the QR code:
→ it opens the DrimPay payment page
→ the customer selects country
→ selects operator
→ enters phone number
→ enters amount (if amount is not fixed)
→ completes payment using the existing payin system.

The QR system must behave like payment links but in QR format.
