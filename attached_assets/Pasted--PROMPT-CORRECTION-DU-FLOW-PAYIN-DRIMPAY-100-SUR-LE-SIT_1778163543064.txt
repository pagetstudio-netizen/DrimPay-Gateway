Permettre aux marchands de :

- collecter des paiements Mobile Money directement depuis leur site

- recevoir les confirmations en temps réel
- utiliser webhooks + polling sécurisé
- éviter les doubles paiements
- avoir une expérience fluide type fintech moderne

---

🌍 FLOW COMPLET DU PAIEMENT

1. Le client est sur le site du marchand
2. Le client entre :
   - numéro téléphone
   - opérateur
3. Le backend marchand appelle DrimPay API
4. DrimPay envoie le prompt Mobile Money
5. Le client confirme sur son téléphone
6. DrimPay traite le paiement
7. DrimPay envoie webhook au marchand
8. Le marchand met à jour la commande automatiquement

Aucune redirection vers une page externe DrimPay.

🔥 CORRECTIONS À AJOUTER

---

1. AJOUTER expires_at

Toutes les transactions doivent expirer automatiquement.

Exemple réponse API :

{
  "reference": "TG-XXXX",
  "status": "pending",
  "expires_at": "2026-05-06T08:35:00.000Z"
}

Durée expiration configurable :

- 2 min
- 5 min
- 10 min

Après expiration :

status = failed
reason = expired

---

2. AJOUTER POLLING FRONTEND

Le marchand doit pouvoir vérifier automatiquement le statut toutes les 3 à 5 secondes.

Exemple :

setInterval(async () => {
  const res = await fetch("/api/check-payment");
}, 5000);

Créer endpoint :

GET /v2/payin/{reference}

Retourner :

- pending
- processing
- success
- failed
- expired

---

3. AJOUTER STATUS COMPLETS

Utiliser :

queued
pending
processing
success
failed
reversed
cancelled
expired

---

4. AJOUTER WEBHOOKS SÉCURISÉS

Tous les webhooks doivent contenir :

X-DrimPay-Signature
X-DrimPay-Timestamp

Utiliser :

- HMAC SHA256
- protection replay attack

Le marchand doit vérifier la signature avant traitement.

---

5. AJOUTER IDPOTENCY COMPLÈTE

Utiliser :

order_id

Si une requête est renvoyée avec le même order_id :

- ne jamais recréer une transaction
- retourner la transaction originale

---

6. AJOUTER RETRY AUTOMATIQUE

Si webhook échoue :

- retry automatique 3 fois
- exponential backoff

Exemple :

- 2 sec
- 4 sec
- 8 sec

---

7. AJOUTER RATE LIMITING

Limiter :

100 req/min/API key

Retour :

{
  "error": "RATE_LIMITED"
}

---

8. AJOUTER WALLET GEO-ISOLATION

Le système doit empêcher :

- cross-country payouts
- retraits depuis mauvais wallet

Exemple :

- fonds TG → retrait TG uniquement

---

9. AJOUTER DASHBOARD API LOGS

Dans dashboard marchand afficher :

- requêtes API
- erreurs
- webhooks
- retries
- temps réponse
- historique paiements

---

10. AJOUTER SANDBOX COMPLET

Créer environnement séparé :

Sandbox :

https://sandbox.drimpay.africa

Live :

https://api.drimpay.africa

Sandbox doit :

- utiliser faux wallets
- faux paiements
- faux webhooks
- faux soldes
